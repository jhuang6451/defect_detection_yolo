# GC10-DET 到 YOLO 格式转换脚本说明文档
---
本文档详细解释了 `convert_voc_to_yolo.py` 脚本的核心逻辑。该脚本的主要功能是将 GC10-DET 数据集（VOC XML 格式，包含嵌套的图片目录）转化为适用于现代 YOLO 系列模型（如 YOLOv5, YOLOv8）的标准格式数据集。

## 1 核心流程概述
---
脚本按照以下几个关键步骤执行：

1. **清理与重构**：创建标准的 YOLO 目录结构，并在每次运行前清理历史残留。
2. **构建图片索引**：遍历所有类别子目录，提取所有图像的绝对路径并建立内存索引（基于文件名）。
3. **XML 解析与坐标转换**：逐一读取 `lables/` 目录下的 XML 标注文件，获取图片尺寸并提取物体边界框（Bounding Box），将其转化为 YOLO 所需的归一化中心点坐标。
4. **类别映射与脏数据过滤**：统一类别命名（中文拼音转英文），并内置了错别字容错机制（如 `8_yahen` 转为正确类别）。过滤掉没有匹配图片的孤立标签文件。
5. **数据集切分与输出**：将清洗后获得的有效「图片-标签」对按照 8:1:1 的比例随机打乱并分配到 `train`、`val` 和 `test` 数据集中，最后生成配置文件 `data.yaml`。

## 2 关键逻辑详解
---
### 2.1 类别映射与错别字修复 (`CLASS_MAP`)
---
原始数据集的标注文件主要采用中文拼音或简写，为了规范模型训练输出，脚本将其统一映射为标准的英文工业缺陷名称。
另外，原始标注在部分 XML 中存在拼写错误（如把 `8_zhadang` 标成了 `8_yahen` 或 `10_yaozhed` 标成了 `10_yaozhe`）。脚本中的 `CLASS_MAP` 字典同时充当了**翻译器**和**纠错器**，确保这些包含错别字的标注也能被正确归类。同时，`UNIQUE_CLASSES` 保证了最终生成的 `data.yaml` 中仅包含正确的 10 个类别。

### 2.2 跨目录图片检索 (`find_images`)
---
原始图片是按类别放置在如 `01_punching_hole` 等不同的子文件夹中的，而所有的 XML 文件全部堆积在 `lables` 文件夹里。
为了高效匹配，脚本首先遍历所有非 `lables` 的子目录，找到所有图片（`.jpg`, `.png` 等），然后构建一个以 `图片文件名` 为键，`完整文件路径` 为值的字典。
当解析 XML 时，通过 `filename` 字段即可在这个字典中以 $O(1)$ 的时间复杂度快速找回图片的原始位置。

### 2.3 XML 解析 (`xml.etree.ElementTree`)
---
脚本利用 Python 标准库 `xml.etree.ElementTree` 解析 VOC 格式的 XML 文件：
- **提取图像尺寸**：从 `<size>` 节点读取 `width` 和 `height`。这是计算相对坐标（归一化）的必备参数。
- **提取边界框**：遍历所有的 `<object>` 节点，读取 `<bndbox>` 内的 `xmin`, `xmax`, `ymin`, `ymax`（绝对像素坐标）。
- **越界截断**：使用 `max()` 和 `min()` 函数对边界框进行裁剪，防止人工标注出现越界（例如 xmin < 0 或 xmax > width）导致 YOLO 训练报错。

### 2.4 VOC 到 YOLO 的数学转换 (`convert_bbox`)
---
VOC 格式使用的是：左上角坐标 + 右下角坐标的绝对像素值 $(X_{min}, Y_{min}, X_{max}, Y_{max})$。
YOLO 格式需要的是：**中心点相对坐标 + 宽高相对比例** $(X_{center}, Y_{center}, W, H)$。

脚本中的 `convert_bbox` 函数执行了以下数学变换：
1. **计算中心点绝对坐标**：
   $X_{abs\_center} = (X_{min} + X_{max}) / 2$
   $Y_{abs\_center} = (Y_{min} + Y_{max}) / 2$
2. **计算绝对宽高**：
   $W_{abs} = X_{max} - X_{min}$
   $H_{abs} = Y_{max} - Y_{min}$
3. **归一化（除以图像宽高）**：
   $X_{yolo} = X_{abs\_center} / Width_{image}$
   $Y_{yolo} = Y_{abs\_center} / Height_{image}$
   $W_{yolo} = W_{abs} / Width_{image}$
   $H_{yolo} = H_{abs} / Height_{image}$

### 2.5 数据切分与物理拷贝
---
脚本在内存中整理出一个包含全部可用数据的列表 `valid_data`（剔除了背景图、找不到对应图片的 XML 以及无法解析的脏数据）。
- **随机打乱**：使用 `random.shuffle()` 以确保数据分布均匀（已固定随机种子 `random.seed(42)` 保证多次运行的切分结果一致）。
- **比例切分**：按照 `80%`, `10%`, `10%` 的长度截取列表，形成训练、验证、测试集。
- **物理输出**：使用 `shutil.copy()` 将图像从分散的原始目录拷贝到集中的 `datasets/GC10-DET_YOLO/images/[split]/`，并在对应的 `labels/[split]/` 目录下生成同名的 `.txt` YOLO 标签文件。

### 2.6 生成 data.yaml
---
数据集构建的最后一步是自动生成符合 Ultralytics 规范的 `data.yaml` 文件，指定了图片存放路径 `path` 以及各类别的具体名称 `names`。有了这个文件，就可以直接启动 YOLO 训练命令。
