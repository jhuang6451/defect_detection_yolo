# 超参数调优指南 (Hyperparameter Tuning Guide)

针对工业零部件小目标缺陷检测（如基于改进 YOLO 算法的场景），我们引入了 SPDConv 和 WIoU-v3。由于小目标的特性以及可能的标注噪声，传统的超参数设置往往不再适用。本指南将指导你如何针对“小目标缺陷检测”微调各项关键超参数。

## 1. 学习率 (Learning Rate, LR) 与调度策略

小目标的特征在深层网络中非常微弱，因此在训练初期的参数更新必须更加谨慎，以避免发生梯度爆炸或破坏好不容易捕获的细微特征。

* **初始学习率 (`lr0`)**: 
  * 建议设置：`0.001` 或更小。
  * 原因：相较于常规物体检测（通常 `0.01`），较小的学习率能让网络在搜索具有挑战性的小目标局部最优解时，步子迈得更稳。
* **最终学习率 (`lrf`)**: 
  * 建议设置：`0.01` (这是相对于初始学习率的乘子，即实际最终为 `lr0 * lrf = 1e-5`)。
* **调度策略 (Scheduler)**:
  * 推荐使用：**余弦退火策略 (Cosine Annealing)** 配合较长的 **Warmup (预热)**。
  * `warmup_epochs`: 建议设为 `3` 到 `5` 个 epoch。
  * `warmup_momentum`: 初始设为 `0.8`。
  * `warmup_bias_lr`: 初始设为 `0.1`。
  * 意义：较长的预热期对于 WIoU-v3 非常关键。它可以在训练初期，在动量尚未稳定时，避免出现极端的预测结果而引发“伪离群点”现象，从而稳定累积平均 IoU (`iou_mean`)。

## 2. WIoU-v3 专有超参数微调 ($\alpha$ 和 $\delta$)

WIoU-v3 引入了非单调聚焦机制（Non-monotonic Focusing Mechanism），其核心思想是根据 Anchor 的“离群度（Outlier Degree）”来分配梯度。具体而言，就是减轻对极端低质量样本（如严重的噪声标注）的过度惩罚，同时优先优化具有代表性的普通质量样本。

这两个关键参数决定了分配梯度的曲线峰值与陡峭程度：
* `alpha` ($\alpha$)：控制非单调聚焦曲线的最高峰位置。
* `delta` ($\delta$)：控制曲线的陡峭程度。

### 调参策略与场景：

**场景 A：数据集标注极其粗糙 / 存在严重边缘模糊 / 存在大量错标噪声**
（此时，网络会将这些难以拟合的脏数据判定为高质量损失即低 IoU）
* **策略**：**减小 $\alpha$，增大 $\delta$**。
* **例如**：设 $\alpha=1.7, \delta=4.0$。
* **原理**：使曲线的峰值“左移”且更“陡”。这意味着提早放弃对低质量样本（大损失区域）的抢救和优化，将注意力全部集中到相对“干净”的数据上。

**场景 B：数据集标注非常精细且规范，且包含大量且集中的小目标缺陷**
（由于目标微小，网络本身难以拟合，因此大多数真实小目标的初始 IoU 都很低）
* **策略**：**保持默认值或适当增大 $\alpha$**。
* **例如**：保持 $\alpha=1.9, \delta=3.0$，或设 $\alpha=2.1, \delta=2.5$。
* **原理**：曲线峰值适度“右移”，或者保持平滑，确保对一般偏困难的小目标（虽然很难，但是标注是正确的！）仍然能够分配到足够的梯度来进行梯度下降学习。

## 3. Anchor 尺寸聚类与 IoU 匹配阈值

### 3.1 Anchor 聚类 (AutoAnchor)

在缺陷检测任务中（例如细长的划痕、极小的沙眼、断裂的边缘），缺陷的长宽比与 COCO 数据集中的通用物体（人、车、狗）截然不同。
* **操作指南**：**必须在你的数据集上进行 Anchor 尺寸的 K-Means 重新聚类。**
* 在 Ultralytics/YOLOv8 框架中，通常在 `train.py` 中会默认开启 `autoanchor=True`，请确保不要关闭它。它会读取你的 `datasets/`，如果发现 COCO 预训练的 Anchor 与当前数据集的 BPR (Best Possible Recall) 小于 0.98，将自动触发重新聚类。

### 3.2 IoU 匹配阈值 (`iou_t` / `box` loss gain)

正负样本匹配时，YOLO 采用一定的 IoU 阈值来判断一个预测框是否命中了真实框 (Ground Truth)。
对于小目标而言，哪怕预测框偏移了仅仅 1 到 2 个像素，它与真实框的 IoU 都会发生断崖式的暴跌！
* **默认阈值**：通常为 `0.5`。
* **针对小目标的微调**：
  * 建议将匹配的 IoU 阈值放宽：**调低至 `0.4` 甚至 `0.35`**。
  * **意义**：如果不降低该阈值，在训练初期，绝大多数微小目标的预测框都无法与 Ground Truth 形成正样本匹配（因为 IoU 达不到 0.5）。这会导致小目标不断被当做“背景（负样本）”来惩罚，最终引发严重的“漏检”问题。

## 4. 数据增强策略微调

为了应对小目标“样本不平衡”以及背景复杂的问题，推荐调整以下增强参数（位于 `hyp.yaml` 或 `train.py` 的参数列表中）：

* **Mosaic 增强 (`mosaic`)**: 强烈建议开启，保持 `1.0`。Mosaic 将四张图片拼接，极大地丰富了检测上下文，同时让目标相对原图更“小”，正好迎合小目标的训练。
* **MixUp 增强 (`mixup`)**: 建议关闭 (`0.0`) 或设得很低 (`0.05`)。因为工业零件背景纹理通常较强，MixUp 会使原本就微弱的小目标特征被过度叠加和掩盖，增加学习难度。
* **复制粘贴 (`copy_paste`)**: 如果可能，对于数量极少的某类极小缺陷，可以尝试开启复制粘贴增强，增加其在画面中出现的频率。

---
*总结：小目标缺陷检测的难点在于“特征微弱”与“边界敏感”。通过放缓学习率、重塑锚框匹配规则、并利用 WIoU-v3 抵抗噪声，可以有效提升模型在复杂工业场景下的鲁棒性与召回率。*
