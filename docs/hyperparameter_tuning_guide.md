# 超参数调优指南 (Hyperparameter Tuning Guide)

针对工业零部件小目标缺陷检测（如基于改进 YOLO 算法的场景），项目引入了 SPDConv、BiFPN 和 WIoU-v3 动态损失函数。由于小目标的特性以及可能的标注噪声，传统的超参数设置往往不再适用。本指南综合了网格搜索（Grid Search）的实证结果，描述了如何针对“小目标缺陷检测”微调各项关键超参数。

## 1. 学习率 (Learning Rate, LR) 与调度策略

小目标的特征在深层网络中非常微弱，加上 WIoU-v3 依赖历史动量，因此在训练初期的参数更新必须更加谨慎，以避免发生梯度爆炸或破坏好不容易捕获的细微特征。

* **初始学习率 (`lr0`)**: 
    - **推荐设置：`0.001`**（相比常规物体检测的 `0.01` 降低了 10 倍）。
    - **原因**：较小的学习率能让网络在搜索具有挑战性的小目标（如极小轧坑、划痕）的局部最优解时，步子迈得更稳。网格搜索结果显示 `0.001` 是本项目的甜点位。
* **最终学习率 (`lrf`)**: 
    - 建议设置：`0.01` (这是相对于初始学习率的乘子，实际最终学习率为 `lr0 * lrf = 1e-5`)。
* **调度策略 (Scheduler)**:
    - 推荐使用：**余弦退火策略 (Cosine Annealing)** 配合较长的 **Warmup (预热)**。
    - `warmup_epochs`: 建议设为 `3.0` 到 `5.0` 个 epoch。
    - `warmup_momentum`: 初始设为 `0.8`。
    - `warmup_bias_lr`: 初始设为 `0.1`。
    - **意义**：较长的预热期对于 WIoU-v3 非常关键。它可以在训练初期，在动量尚未稳定时，避免出现极端的预测结果而引发“伪离群点”现象，从而稳定累积平均 IoU (`iou_mean`)。

## 2. WIoU-v3 专有超参数微调 ($\alpha$ 和 $\delta$)

WIoU-v3 (Wise-IoU version 3) 引入了非单调聚焦机制（Non-monotonic Focusing Mechanism），可以自动抑制质量极差的异常标注（比如边缘极度模糊的划痕、错误标注的油污点等）。

这两个关键参数决定了分配梯度的曲线峰值与陡峭程度：
* **$\alpha$ (Alpha)**：控制非单调聚焦曲线的最高峰位置。决定了什么级别的离群样本应该受到多大的关注。
* **$\delta$ (Delta)**：控制曲线的陡峭程度（降权系数斜率的力度）。控制抑制效果的剧烈程度。

### 🌟 实证结果：GC10-DET 网格搜索最佳参数
经过大量实验比判，在工业缺陷检测数据集（GC10-DET）上，**最佳参数组合**为：
* **$\alpha = 1.6$**
* **$\delta = 2.5$**

**原理解析：**
1. **为什么较小的 $\alpha$ (1.6) 表现最好？**
   在含有较多边缘模糊、错标目标的工业数据集中，较小的 $\alpha$ 意味着模型**更早地“放弃”对低质量样本（如噪声标注）的抢救**，将梯度集中优化在能够拟合的特征上。
2. **为什么 $\delta=2.5$ 是甜点位？**
   $\delta=3.0$ 能带来极致的 Precision（压制大量误报），但 $\delta=2.5$ 在不显著损失 Precision 的前提下，极大地保障了高召回率（Recall），使 mAP 达到最高。

## 3. Anchor 自动聚类与 IoU 匹配阈值

诸如钢带边缘破损、狭长划痕等缺陷，其感受野的长宽比与 COCO 等自然数据集差异极大。

* **Anchor 先验聚类 (AutoAnchor)**：
  在 Ultralytics/YOLOv8 框架中，通常默认开启 `autoanchor=True`。它会读取你的数据集，如果发现 COCO 预训练的 Anchor 与当前数据集不够匹配，将自动触发重新聚类（K-Means）。**请确保不要关闭它**。
* **放宽匹配阈值 (`iou_t` / `box` loss gain)**：
  对于小目标而言，哪怕预测框偏移了仅仅 1 到 2 个像素，它与真实框的 IoU 都会发生断崖式的暴跌！
  * **建议**：如果是早期版本，可以调整标签分配策略。在现代 YOLO 框架（如 v8 的 TaskAlignedAssigner）中，该分配已高度自适应，但我们可以在 `hyp.yaml` 中适当上调 `box` 损失系数，强化定位回归。

## 4. 数据增强策略微调

为了应对小目标“样本不平衡”以及背景复杂的问题，推荐以下增强策略：

* **Mosaic 增强 (`mosaic`)**: **强烈建议开启，保持 `1.0`**。Mosaic 将四张图片拼接，极大地丰富了检测上下文，同时让目标相对原图更“小”，正好迎合小目标的训练。
* **MixUp 增强 (`mixup`)**: **建议关闭 (`0.0`) 或设得很低 (`0.05`)**。因为工业零件背景纹理通常较强，MixUp 会使原本就微弱的小目标特征被过度叠加和掩盖，增加学习难度。
* **漏检感知 Copy-Paste (Leakage-Aware Copy-Paste)**: 针对数量极少、漏检率极高的严重不平衡类别（如 `rolled_pit`, `crease`），使用本项目 `datasets/augment_dataset.py` 生成的独立增强数据集进行专项弥补。

---
*总结：小目标缺陷检测的难点在于“特征微弱”与“边界敏感”。通过放缓学习率（`0.001`）、使用前置融合网络（`BiFPN`）并利用 `WIoU-v3 (α=1.6, δ=2.5)` 抵抗噪声，可以有效提升模型在复杂工业场景下的鲁棒性与召回率。*
